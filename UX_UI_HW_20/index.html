CWSDS Website Redesign

Case Study


PHASE 1: User Research.

After user interviews and heuristic evaluation, a few things were noticed about the old website..

-Contrast ratio too low in header nav.

-Breadcrumbs lacking

-Hero image very large, prominent, but not CTA

-A lot of content space being taken up by Icon Cards. Possible to economize space. The logos are too big

-Text is too small and can be placed outside the image

-The buttons are too similar to the body text

-The footer needs to be more focused

-Need a Search Bar

-Need to fix the Donation button contrast

PHASE 2: Definition and Synthesis

A problem was then defined: A mother needs to find a safe place for her four year old son who is living with developmental challenges amidst her and her partner’s hectic work schedule. How might we help her get access to the information on the services she needs for her son/daughter with developmental challenges? How might we help CWSDS gain brand awareness and more donor conversions by making the ‘donate’ user flow more visible and accessible?

PHASE 3: Ideation

We created a new sitemap and then formulated new wireframes. We created a fitting style guide that people with developmental disabilities might think is more 'fun' and attractive.


PHASE 4: Prototyping.

Using the style guide and new sitemap, we created a more attractive version of the CWSDS website.

PHASE 5: User Testing

Five geurilla user tests were conducted. We received feedback that there was too much scrolling and that the 'Donate' button should also be on the Header for easier navigatioin. We iterated our prototypes based off of this feedback.

PHASE 6: Conclusion

In the future, we could spend more time experimenting with the UI tools. We spent more time on the details of designing than trying to use more UI elements. We should have also kept an UX journal of all the micro-decisions that we made.
